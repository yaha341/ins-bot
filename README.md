# 📸 Instagram Bot - Автоответы по кодовым словам

Автоматический бот для Instagram, который отслеживает комментарии под вашими постами и автоматически отвечает в комментариях + отправляет Direct сообщения при обнаружении кодовых слов.

## 🎯 Возможности

- ✅ **Автоматическое отслеживание комментариев** под вашими постами/reels
- 🔑 **Триггеры по кодовым словам** - настраиваемые правила для каждого поста
- 💬 **Автоответ в комментариях** - мгновенная реакция на комментарий
- ✉️ **Отправка Direct сообщений** - автоматическая рассылка в личные сообщения
- 🚫 **Fallback механизм** - если DM не доставлен (закрытый аккаунт), бот ответит в комментариях
- 📊 **Логирование всех срабатываний** - полная история триггеров
- 🎨 **Современная админ-панель** - удобное управление правилами
- 🔒 **Безопасность** - авторизация в админке, защита API endpoints
- ⚡ **Serverless архитектура** - работает на Vercel без своего сервера

## 🏗️ Архитектура

```
┌─────────────┐
│   Vercel    │ ← Frontend (React + Vite)
│  Serverless │ ← Backend API (TypeScript)
└──────┬──────┘
       │
       ├─── Vercel Cron (каждые 7 минут)
       │    └─→ /api/poll - проверяет новые комментарии
       │
       ├─── Instagram Private API
       │    └─→ Авторизация, комментарии, DM
       │
       └─── Supabase (PostgreSQL)
            ├─ ig_rules - правила автоответов
            ├─ ig_triggers - лог срабатываний
            ├─ ig_processed_comments - дедупликация
            └─ ig_settings - сессия Instagram
```

## 🚀 Быстрый старт

### 1. Подготовка Supabase

1. Создайте проект на [supabase.com](https://supabase.com)
2. Перейдите в SQL Editor
3. Выполните скрипт из `supabase-schema.sql`
4. Сохраните:
   - **Project URL** (SUPABASE_URL)
   - **Service Role Key** (SUPABASE_SERVICE_ROLE_KEY)

### 2. Деплой на Vercel

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/yaha341/ing-bot)

**Или вручную:**

```bash
# Клонируйте репозиторий
git clone https://github.com/yaha341/ing-bot.git
cd ing-bot

# Установите зависимости
npm install

# Деплой на Vercel
npx vercel

# Настройте переменные окружения (см. ниже)
npx vercel env add
```

### 3. Настройка переменных окружения

В настройках проекта Vercel добавьте:

```env
# Supabase
SUPABASE_URL=https://xxxxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=eyJhbGci...

# Instagram (основной аккаунт бота)
IG_USERNAME=ваш_логин_инстаграм
IG_PASSWORD=ваш_пароль

# Админ-панель
ADMIN_LOGIN=admin
ADMIN_PASSWORD=ваш_надежный_пароль
ADMIN_SESSION_SECRET=случайная_строка_32_символа

# Cron защита (опционально)
CRON_SECRET=случайная_строка_для_защиты_cron
```

⚠️ **Важно:**
- После добавления переменных сделайте redeploy проекта
- Используйте надёжные пароли для `ADMIN_PASSWORD`
- `ADMIN_SESSION_SECRET` - минимум 32 случайных символа

### 4. Первый запуск

1. Откройте `https://ваш-проект.vercel.app`
2. Войдите с логином и паролем из `ADMIN_LOGIN` / `ADMIN_PASSWORD`
3. Перейдите в **⚙️ Настройки**
4. Подключите Instagram аккаунт (введите логин, пароль берётся из `IG_PASSWORD`)
5. Создайте первое правило в **📋 Правила**

## 📝 Создание правила

1. Скопируйте ссылку на пост/reels из Instagram
2. Нажмите **+ Добавить правило**
3. Вставьте ссылку и нажмите **🔍 Определить** (Media ID подтянется автоматически)
4. Добавьте кодовые слова (например: `подарок`, `хочу`, `участвую`)
5. Напишите текст для Direct сообщения
6. Настройте автоответы в комментариях
7. Сохраните

**Пример правила:**
```
Пост: https://www.instagram.com/reel/ABC123/
Кодовые слова: подарок, хочу, участвую
Ответ в комментарий: "Ответила вам в личные сообщения 😊"
Direct: "Привет! Вот ссылка на материалы: https://example.com"
Fallback: "Не смогла отправить сообщение — у вас закрытый аккаунт..."
```

## 🔧 Как это работает

### Механизм Polling (автоматический)

1. **Vercel Cron** каждые 7 минут вызывает `/api/poll`
2. API загружает все активные правила из Supabase
3. Для каждого Media ID получает последние комментарии через Instagram Private API
4. Проверяет каждый комментарий на наличие кодовых слов
5. При совпадении:
   - Отвечает в комментарий
   - Пытается отправить DM
   - Если DM не доставлен → отправляет fallback комментарий
   - Логирует триггер в базу
   - Помечает комментарий как обработанный

### Альтернатива: Webhook (требует Facebook App)

В проекте есть `/api/webhook`, но для его работы нужно:
1. Создать Facebook Business App
2. Пройти App Review от Meta
3. Настроить Webhooks для Instagram

**Рекомендация:** используйте Polling (уже настроен), webhook оставьте для масштабирования.

## ❓ FAQ

**Q: Бот не отвечает на комментарии?**

1. Проверьте статус подключения в **⚙️ Настройки**
2. Убедитесь что правило **активно** (зелёная галочка)
3. Проверьте что кодовое слово точно совпадает (регистр не важен)
4. Посмотрите логи в Vercel Functions → `/api/poll`

**Q: "Не удалось подключиться к Instagram"**

- Проверьте логин/пароль в переменных окружения
- Instagram может потребовать подтверждение через email/SMS при первом входе
- Попробуйте войти с этого IP через браузер сначала
- Если используете 2FA - отключите временно

**Q: Как часто проверяются комментарии?**

- Каждые 7 минут (настроено в `vercel.json`)
- Можно изменить на `*/5 * * * *` (каждые 5 минут) или `*/10 * * * *` (каждые 10 минут)
- После изменения сделайте `git push` для применения

**Q: Можно ли использовать для нескольких аккаунтов?**

- Текущая версия работает с одним Instagram аккаунтом
- Для множественных аккаунтов нужно доработать схему БД и логику авторизации

**Q: Бот отвечает на старые комментарии?**

- Нет, при первом запуске все существующие комментарии помечаются как обработанные
- Бот реагирует только на новые комментарии после создания правила

**Q: Лимиты Instagram API?**

- Instagram может ограничить активность при большом объёме запросов
- Рекомендуется не более 20-30 правил одновременно
- Используйте задержки между действиями (уже реализовано в коде)

## 🐛 Troubleshooting

### "Error: Missing required env vars"
→ Добавьте все переменные окружения в Vercel и сделайте redeploy

### "Error: instagram-private-api login failed"
→ Проверьте учётные данные, отключите 2FA, попробуйте войти через браузер

### "Error: Media ID not found"
→ Убедитесь что пост публичный и ссылка корректная

### Комментарии не обрабатываются
→ Проверьте что Vercel Cron активен (в настройках проекта)
